# 梧桐墙：校园社区
一种适配移动端的类校园集市社区，以FastAPI + SQLite + 原生 JS 实现的校园社区全栈原型，包含树洞、游戏车队、二手集市、问答、课评、活动、私信、公告与简陋管理员后台。

## 目录结构

```
campus-wall/
├── backend/
│   ├── main.py          # 核心：认证 / 树洞 / 车队 / 集市 / 信用 / 热榜 / 提醒调度
│   └── extras.py        # 扩展：问答 / 手册 / 课评 / 观察台 / 治理 / 失物 / 活动 / 私信 / 公告 / 举报
├── frontend/
│   ├── index.html       # 前端单页（自适应手机端；不启动后端时自动降级为静态演示）
│   ├── app-extra.js     # 扩展板块的动态渲染与发布表单
│   ├── polish.js        # 交互与动效增强层
│   └── app-v4.js        # v4：编辑器/回帖/车队增强/管理员后台/用户后台
├── requirements.txt
└── README.md
```

## 运行

```bash
cd campus-wall/backend
pip install -r ../requirements.txt
python -m uvicorn main:app --reload --port 8000
```

- http://127.0.0.1:8000 → 完整网站
- http://127.0.0.1:8000/docs → 交互式 API 文档（40+ 接口可直接调试）

首次启动自动建库并写入演示数据。数据库默认在 `backend/campus.db`，可用环境变量 `WUTONG_DB` 覆盖路径。

## 管理员操作（用 /docs 或前端）

观察台过审：`POST /api/observe/{id}/review`；手册加精：`POST /api/handbook/{id}/feature`；课评更正代提交：`POST /api/courses/review/{id}/correction`。

## 接真实邮件

`main.py` 的 `notify()` 目前写入 `email_log` 表（`GET /api/emails` 可查看）。接入真实邮箱把写表部分替换为 `smtplib` 发送即可（QQ 邮箱 SMTP：smtp.qq.com:465，需生成授权码）。

## 部署提示

单校规模 SQLite 足够，并发上来换 PostgreSQL。生产环境务必：HTTPS、给 `/api/emails` 和管理员接口加严格鉴权、接入完善敏感词库、加验证码与频率限制、日志与备份。
